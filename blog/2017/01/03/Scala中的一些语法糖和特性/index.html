Scala学习笔记（一）- Scala的一些语法糖和特性

编程语言Scala
  • functional programming
  • scala

引言

最近在学习scala，看了不少书，觉得这个语言很好，很对我的口味，也是我第一次接触函数式编程，有的地方还不是非常理解，所以记录一些特性，包括语法糖，和java的联系与区别，会不断更新。

函数式编程理念

函数式编程有两种理念，第一种理念是函数是头等值，第二种理念是程序的操作应该把输入值映射为输出值，而不是就地的修改数据，第二种理念的另一种解释是方法不应该有任何副作用，方法与其所在的环境的交流的唯一方式应该是获得参数和返回结果，就是说对于任何输入来说，都可以用方法的结果替代对它的调用，而不影响程序的语义。

结果类型

Java的返回类型在Scala里被称为结果类型。如果函数是递归的，那么方法的结果类型必须被明确的说明。

没有运算符重载

Scala里没有运算符重载，取而代之的是诸如+，-，/，*这样的字符，可以用作方法名。

apply方法

用括号传递给变量一个或多个值参数时，Scala会把它转换成对apply方法的调用。

update方法

当对有括号并包括一到若干参数的变量赋值时，编译器将使用对象的update方法对括号里的参数（索引值）和等号右边的对象执行调用。 str(0) = "Hello"被转换成str.update(0, "Hello")

冒号结尾方法

方法名以冒号结尾，方法被右操作数调用，例如１ :: example就是example.::(1)

Nil和cons

因为Nil是空列表的简写，所以可以使用cons(::念做cons)操作符把所有元素都串起来，并以Nil结尾来定义新列表。

输出　List(1, 2, 3)

访问元组

为什么不能用访问列表的方法来访问元祖，如pari(0)，因为列表的apply方法始终返回同样的类型，而元祖里的类型不尽相同，所以只能用_1,_2来访问。

可变集和不可变集的+方法

可变和不可变的集都提供了“＋”方法，但可变集把元素添加到自身，而不可变集则创建并返回添加了元素的新集。

去掉方法体的等于号

如果去掉方法体前面的等号，那么方法的结果类型必定是Unit，这种说法不论方法体内包含什么都成立，因为Scala编译器可以把任何类型都转换成Unit，而结果类型为Unit的方法来说，执行的目的就是为了它的副作用。

伴生对象

当单例对象与某个类共享一个名称时，它就被称为这个类的伴生对象(companion object)，类和它的伴生对象必须在一个源文件里，类被称为这个单例对象的伴生类，类和它的伴生对象可以互相访问其私有成员。

App特质

Application特质：首先在单例对象后加上extends Application，然后就不需要写main方法，只需把想要执行的代码直接放在单例对象的花括号之间，能这么做，是因为特质Application声明了带有合适签名的main方法，并被你写的单例对象继承。花括号之间的代码被收集进了单例对象的主构造器，并在类被初始化时执行。

忽略转义

以三个引号（"""）作为开始和结束。内部的原是字符串可以包含无论何种任意字符，包括新行、引号和特殊字符，当然同一行的三个引号除外。如果想去掉每行的前导空格，可以把管道符号（｜）放在每行的开头，然后对整个字符串调用stripMargin方法。

操作符

Scala里的操作符不是特殊语法，任何方法都可以是操作符。

无参方法

Scala里，方法调用的空括号可以省掉，惯例是如果方法带有副作用就加上括号，如println();如果没有副作用就去掉括号，如String的toLowerCase方法。

富包装器

每个基本类型都对应着一个”富包装器“提供许多额外的方法，如Int对应着scala.runtime.RichInt

不可变对象

不可变对象的优点：

  • 它们没有随着时间变化的复杂的状态空间。
  • 可以很自由地传递不可变对象
  • 一旦不可变对象完成构造后，就不会有线程因为并发访问而破坏对象内部状态，因为根本没有线程可以改变对象的状态。
  • 不可变对象让哈希表键值更安全，例如如果可变对象进入HashSet后被改变，那么你下一次查找这个HashSet时就找不到这个对象了。

不可变对象的缺点:

  • 有时需要复制很大的对象表而可变对象的更新可以在原址发生。

面向对象

面向对象编程的优点之一是它允许你把数据封装在对象之内以确保数据在整个生命周期中的有效性。

require方法

先决条件是对传递给方法或构造器的值的限制，是调用者必须满足的需求。一种方式是使用require方法，它带一个布尔型参数，如果传入值为真，require方法讲正常返回，否则抛出。

常量名

Scala里习惯常量名只是第一个字母必须大写，常数也是驼峰风格。

if可以返回值

Scala的if是能返回值的表达式，如val filename = if (!args.isEmpty) args(0) else "default.txt"

for表达式里的过滤器

for表达式里可以添加过滤器，即if子句，当发生器中过滤器超过一个时，if子句中间需要以分号隔开。

for表达式里的流间变量绑定

在for表达式里进行流间(mid-stream)变量绑定，绑定的变量被当做val引入和使用，当前面不需要加val关键词。

match可以产生值

与Java的switch相比，Scala的match也可以产生值。

内部变量

java不允许在内部范围内创建与外部范围变量同名的变量，在Scala里，内部变量被认为是遮蔽了同名的外部变量，因为在内部范围内，外部变量变得不可见。

头等函数

Scala的函数是头等函数(first-class function)，你不仅可以定义和调用函数，还可以把它写成匿名的字面量(literal)，并把它们作为值进行传递。函数字面量被编译进类，并在运行期间被实例化为函数值。因此函数字面量和值的区别在与函数字面量存在于源代码中，而函数值作为对象存在与运行期。

FunctionN特质

任何函数值都是某个扩展了Scala包的若干FuncitionN特质之一的实例，如Function0是没有参数的函数，Function1是有一个参数的函数等等。每个FunctionN特质有一个apply方法来调用函数。

传名参数

传名参数，要实现一个传名参数，要定义参数的类型开始于=>而不是()=>。例如，你可以通过改变其类型，把“()=>Boolean”改为“=>Boolean”，传名类型中，空的参数列表，()被省略，它仅在参数中被允许，没有什么传名变量或传名字段的东西。还有括号里的表达式并不是先于方法的调用被评估的，而是代之以先创建一个函数值，其apply方法将评估括号里的表达式，而这个函数值将被传递给方法。

抽象方法

Scala里一个方法只要没有没有实现（即没有括号或方法体），它就是抽象的，并不需要像Java里的abstract修饰符。

无参方法和空括号方法

无参数方法def width(): Int与空括号方法def width: Int是一样的，按照惯例，只要方法中没有参数并且方法仅能通过读取所包含对象的属性去访问可变状态（特指方法不能改变可变状态），就使用无参数方法。这个惯例支持统一访问原则，就是说客户代码不应由属性是通过字段实现还是方法实现而受到影响。例如每个实现里的def换成val。

访问字段

访问字段比调用方法略快，因为字段值在类初始化的时候就被预计算，而方法调用在每次调用的时候都要计算。另一方面，使用字段需要为每个对象分配更多的空间，因此一个属性是用字段表示更好还是方法表示更好与类的使用情况有关。

省略空括号

原则上，Scala的函数调用中可以省略所有的空括号。然而，在调用的方法超出其调用者对象的属性时，仍推荐写一对空的括号。例如，如果方法执行了I/O，或写入可重新赋值的变量var，或读取不是调用者字段的var，总之无论是直接还是非直接的使用可变对象，都应该添加空括号。

Scala鼓励使用将不带参数且没有副作用的方法定义为没有括号方法的风格，但是永远不要定义没有括号但是有副作用的方法，因为那样的话方法调用看上去会像在使用字段，让你的客户会很奇怪为什么它带有副作用。

AnyRef

如果省略extends语句,Scala编译器将隐式地假设你的类扩展自scala.AnyRef,这与java平台上的java.lang.object相同。

命名空间

Scala一般来说仅有两个命名空间

  • 值(字段，方法，包，单例对象)
  • 类型(类，特质名)

所以字段可以重写无参数方法，而且Scala里禁止在同一个类里用同样的名称定义字段和方法。

override

Scala要求，若子类成员重写了父类的具体成员则必须带有override修饰符，若成员实现的是同名的抽象成员时，则这个修饰符是可选的。

==和!=不能被重写

Any类里的等号和不等号方法、==和!=,被声明为final,因此它们不能在子类里重写。实际上，==总是与equal相同，!=总是与equals相反，因此独立的类可以通过重写equals方法改变==或!=的意义。

AnyVal

AnyVal是Scala里每个内建值类的父类（Byte,Short,Char,Int,Long,Float,Double,Boolean,Unit）,值类都是被定义为既是抽象又是final的。

==是自然相等的，eq是引用相等的

Scala里的相等操作==被设计为表达透明,对值类型来说，就是自然的（数学或布尔）相等，而Java中==被设计成引用相等。Scala里AnyRef类定义了附加的eq方法，它不能被重写并且实现为引用相等，作用相当于Java中的==那样，同样也有一个eq的反义词，被称为ne。

Null类

Null类是null引用对象的类型，它是每个引用类（就是说，每个继承自AnyRef的类）的子类。Null不兼容值类型。例如不能把null赋给整数变量。而Nothing类型在Scala的类层级的最底端，它是任何其它类型的子类型。 可以这样灵活的使用Nothing

类和特质的差别，特质是动态绑定的

类和特质的差别在于不论在在类的哪个角落，super调用都是静态绑定的，而在特质中，它们是动态绑定的，在你定义特质的时候super调用的方法实现尚未被定义，调用的实现将在每一次特质被混入到具体类的时候才被决定。这种处理super的行为使得特质能以可堆叠的改变方式工作的关键 如下为例子

特质不是多重继承

特质并不是多重继承，对于多重继承来说，super调用导致的方法调用可以在调用发生的地方明确决定。而对于特质来说，方法调用是由类和被混入到类的特质的线性化所决定的，当混入多个特质时，越靠近右边的特质越先起作用。

特质与抽象类

如果计划以编译后的方式发布它，并且你希望外部组织能够写一些继承自它的类，你应该更倾向于使用抽象类，因为当特质获得或失去成员，所有继承自它的类就算没有改变也都要被重新编译。
如果效率非常重要，则应该更倾向于使用类，大多数Java运行时都能让类成员的虚方法调用快于接口方法调用，特质被编译成接口，因此会付出微小的性能代价。

访问限定词

Scala里的访问修饰符可以通过使用限定词强调，格式为private[x],protected[x]，表示“直到”的私有或保护，这里x指代某个所属的包、类或单例对象。 Scala还有一种比private更严格的访问修饰符。被private[this]标记的定义仅能在包含了定义的同一个对象中被访问，这种定义被称为对象私有。把成员标记为private[this]可以保证它不能被同一个类中的其他对象访问。

伴生对象成员

类的所有访问权限都伴生对象开放，反过来也是如此。伴生对象的成员没有意义，因为单例对象没有任何子类。

样本类在语法上的便捷设定

  • 它会添加与类名一致的工厂方法。
  • 样本类参数列表的所有参数隐式获得了val前缀，因此它被当做字段维护。
  • 编译器为你的类添加了方法toString、hashcode、equals的“自然”实现。
  • 样本类支持模式匹配。

模式的种类

  • 通配模式：通配模式（_）匹配任意对象

  • 常量模式：常量模式只匹配自身

  • 变量模式：变量模式类似于通配符，可以匹配任意对象

  • 构造器模式：加入该名称指定了一个样本类，那么这个模式就是表示首先检查对象是该名称的样本类成员，然后检查对象的构造器参数是符合额外提供的模式的。这些额外的模式意味着Scala模式支持深度匹配，这种模式不止检查顶层对象是否一致，还会检查对象的内容是否匹配内层的模式。 如

  • 元祖模式：可以匹配元祖

  • 类型模式：可以把类型模式当做类型测试和类型转换的简易替代

泛型擦除

Scala使用了泛型的擦除模式，就如一样，也就是说类型参数信息没有保留到运行期，因此运行期没有办法判断给定的Map对象创建时带了两个Int参数还是其他的什么类型。擦除规则的唯一例外就是数组，Java和Scala里，它们都被特殊处理了，数组的元素类型和数值保存在一起，因此它们可以做模式匹配。

模式守卫

模式守卫接在模式之后，守卫可以是任意的引用模式中变量的布尔表达式。

全匹配

全匹配的样本要跟在更具体的简化方法之后，如果写成其他次序，那么全匹配样本将比特定规则获得更高的优先级，在许多情况下，编译器将在你做这种尝试时发出警告。

封闭类

封闭类除了类定义所在的文件之外不能在添加任何新的子类。这对于模式匹配来说是非常有用的，因为这意味着你仅需关心你已经知道的子类即可。这还意味着你可以获得更好的编译器帮助，如果你使用继承自封闭类的样本做匹配，编译器将通过警告信息标识出缺失的模式组合。

Option类型

Scala为可选值定义了一个名为Option的标准类型。这种值可以有两种形式，可以是some(x)的形式，其中x是实际值，也可以是None对象，代表缺失的值。Scala集合类的某些标准操作会产生可选值，例如Map的get方法会在发现了指定键的情况下产生some(value)，在没有找到指定键的时候产生None对象，代表缺失的值。 分离可选值最通常的方法就是通过模式匹配，例如

可用通过使用模式拆分元祖并把其中的每个值分配给变量

Nil和cons

所有列表都是由两个基础构造块Nil和::构造出来的。

不可变集的+=操作

纵使不可变集和映射并不支持真正的+=方法，Scala还是为此提供了+=的语法解释。如果你写了a += b，而a不支持+=的方法，Scala将尝试把它解释为a= a+b，因此变量必须是var类型，不能为val类型。

想要把当前使用的可变集合转换成不可变集合，可以先创建空不可变集合，然后把可变集合的元素用++操作符添加进去。

getter和setter

var变量x的getter方法命名为“x”,它的setter方法命名为“x_=”。

下界与上界

下界　U >: T ，定义了T为U的下界，U必须是T的超类型 上界　T <: U ，定义了U为T的上界，T必须是U的子类型

里氏替换原则

里氏替代原则（LSP)：如果你能在需要类型U的地方替换成类型T的话，那么类型T是类型U的子类型的假设就是安全的。

预初始化字段

预初始化字段，可以让你在调用超类之前初始化子类的字段。操作的方式是把字段定义加上花括号，放在超类构造器调用之前。预初始化字段不仅限于匿名类，还可以被用于对象或有名称的子类，预初始化段落必须被放在定义的对象或类的extends关键字之后。

# 惰性计算 如果你把lazy修饰符前缀在val定义上，那么右侧的初始化表达式将直到val第一次被使用的时候才计算。与def不同的是，懒加载的val绝对不会计算第二次，实际上第一次计算懒加载val的时候结果就被保存了下来，以备同样的val后续使用。懒加载val可以免去你作为程序员不得不考虑的问题，即如何val定义顺序，以确保所有东西在需要的时候已经完成定义。

temp: A#B 该参数指明temp必须是某种未知但任意的A的B类型

implicit关键字是应用于全体参数列表，而不是单独的参数。

视界，T <% Ordered[T]表示，任何的T都好，只要T能被当作Ordered[T]即可
